## 🐍 Проверка Python окружения

### Windows (PowerShell)

```powershell
# 1. Проверить Python установлен
python --version
# ✅ Должно вывести Python 3.10+

# 2. Перейти в папку проекта
cd d:\WebSite\university_catalog

# 3. Создать виртуальное окружение
python -m venv venv

# 4. Активировать окружение
venv\Scripts\activate
# ✅ Должно показать (venv) в начале строки

# 5. Установить зависимости
pip install -r requirements.txt

# 6. Проверить установку
pip list | Select-String django,pillow
```

### Mac/Linux (Bash)

```bash
# 1. Проверить Python установлен
python3 --version
# ✅ Должно вывести Python 3.10+

# 2. Перейти в папку проекта
cd ~/university_catalog  # Или ваш путь

# 3. Создать виртуальное окружение
python3 -m venv venv

# 4. Активировать окружение
source venv/bin/activate
# ✅ Должно показать (venv) в начале строки

# 5. Установить зависимости
pip install -r requirements.txt

# 6. Проверить установку
pip list | grep -E "django|Pillow"
```

## 🐛 Решение проблем

### Проблема: "No module named 'django'"

```bash
pip install -r requirements.txt
# Или
pip install Django==5.0.3 Pillow==10.2.0
```

### Проблема: "No module named 'PIL'"

```bash
pip install Pillow
```

### Проблема: Сервер не запускается

```bash
# Убедитесь что:
# 1. Виртуальное окружение активировано
# 2. Все зависимости установлены
# 3. БД создана (migrate)

python manage.py migrate
python manage.py runserver
```

### Проблема: Админка не загружается

```bash
# Убедитесь что:
# 1. Суперпользователь создан
# 2. Все миграции применены
# 3. /admin/ в urls.py

python manage.py createsuperuser
python manage.py migrate
python manage.py runserver
```

### Проблема: Логотипы не отображаются

1. Проверьте что DEBUG = True
2. Проверьте что папка media существует
3. Проверьте права доступа:

   ```bash
   # Windows PowerShell
   icacls "media" /grant:r "%USERNAME%:F"

   # Mac/Linux
   chmod -R 755 media
   ```
